Карта развития

Текст дальше является не картой, а списком, которого я буду первое время придерживаться и по которому составлять вопросы для изучения на следующий день.

Сам список

  1. HTML — знание основных тегов, аттрибутов. Понимание, как его писать в принципе.
  2. Типографика — умение отформатировать текст. Текст — основа практически любого проекта. Вставка неразрывных пробелов где надо, выделение жирным, курсивом, abbr и так далее. Можно использовать типограф или схожий сервис, но уметь понимать результат.
  3. Семантика — понимание, что для определённых задач есть определённые теги. Уметь выбрать нужный тег.
  4. Медиа — какие виды медиа можно встраивать в страницу.
  5. Iframe — встраивание сторонних виджетов (видео, аудио, карты и т.п.).
  6. Аудио, Видео — можно отложить изучение, т.к. частично решаются с помощью iframe. Понимать, какие форматы умеет воспроизводить браузер, как оформить плеер и так далее.
  7. Картинки — какие графические форматы и в каком виде воспринимает браузер. Плюсы и минусы использование тех или иных форматов.
  8. Растр — jpg, png, gif. Понимать различие форматов и уметь применять что и где надо.
  9. SVG — можно отложить изучение, т.к. применяется реже, чем хотелось бы. Понимать плюсы, минусы, ограничения и т.п.
  10. Шрифты — можно отложить изучение. На самом деле довольно сложная тема, и в общем случае я бы советовал начинающим использовать системные шрифты. Уметь подгрузить шрифты, оптимизировать отображение, минимизировать лаг отрисовки с кастомными шрифтами.
  11. Табличная вёрстка — опционально. Для желающих верстать качественные email-рассылки в будущем.
  12. CSS 1 — шрифты, цвета, выравнивание, размеры.
  13. CSS 2.1 — управление поведением блоков, позиционирование, полноценное оформление.
  14. Селекторы — простые селекторы на тег, класс, вложенный элемент. Простые псевдоселекторы типа :hover.
  15. Именование — как называть классы, чтобы не было мучительно больно.
  16. Блочная вёрстка — разбить картинку на осознанные блоки, воплотить блоки в HTML, оформить с CSS.
  17. Браузеры — можно отложить изучение. Какие браузеры существуют, в чём их различие.
  18. Инструменты разработчика — можно отложить изучение. Использовать инструменты браузеров, чтобы разбираться в проблемах отображения.
  19. Текстовые редакторы — какие текстовые редакторы для разработчиков есть и зачем. SublimeText и Notepad++ приведены для примера, как хорошо мне знакомые. Уметь настраивать в них базовые вещи, такие как отступы, перенос строки и так далее.
  20. Системы контроля версий — я лично считаю умение их использовать хотя бы индивидуально, лично для себя, очень важным. Понимать, для чего эти системы созданы и какие бывают.
  21. Git — понимать в общих чертах задачу и принципы самой популярной системы контроля версий.
  22. Github / bitbucket — уметь использовать одну из популярных площадок для git.
  23. Checkout / commit / push / pull — базовые операции для личного использования.
  24. Stash — для временного сохранения ненужных в данный момент данных.
  25. 10 работ — сделать минимум 10 работ в различном дизайне. Можно тестовых, это не важно. Важно, чтобы полноценных в рамках текущих знаний.
